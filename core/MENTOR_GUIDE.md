# EasyMentor - Mentor Guide

## 1) Login
- Open web: `https://easymentor-web.onrender.com/`  
  (or localhost URL if testing locally).
- Enter mentor username/password.
- You can login using full mentor name or short name (mapped to same mentor account).

## 2) Select Correct Module
- At top bar, select the correct module (example: `FY2 - Batch 2026-29_Sem-1`).
- All data is module-wise. If wrong module is selected, you may see empty or wrong records.

## 3) Mentor Menu
- `Attendance Calls`
- `Result Calls`
- `Direct Calls`
- `PRINT SIF`
- `Attendance Report`
- `Result Report`

---

## 4) Attendance Calls (Teaching Period)
### What you see
- Students who require attendance follow-up (as per coordinator upload).
- Status colors:
  - Blue = Pending
  - Yellow = Not Received
  - Green = Received

### How to work
1. Open selected week.
2. Tap/click call button.
3. After call, fill popup:
   - Talked with
   - Duration
   - Parent remark
4. Mark `Received` or `Not Received`.
5. If not received, use message option where needed.

---

## 5) Result Calls (Exam Period)
### What you see
- Only poor-result based call list (separate from attendance calls).

### How to work
1. Open test+subject cycle uploaded by coordinator.
2. Call student parent.
3. Fill popup details and mark `Received` / `Not Received`.
4. Send message if call not received.

---

## 6) Direct Calls
Direct Calls has 3 categories in popup:
- `Less Attendance`
- `Poor Result`
- `Other`

### A) If category = Less Attendance
- Enter:
  - Week Number
  - Day Number
- Prefill values:
  - Parents remark: `Sick`
  - Faculty reason: `Absent on WK-{week} DAY-{day}`
- This entry is also written into attendance record and visible in SIF attendance section.

### B) If category = Poor Result
- Enter:
  - Name of Exam
  - Subject Name
  - Marks Obtained
  - Out of Marks (25/50/100)
- Parent remark prefill: `Student will Study more`
- This entry appears in SIF Poor Result section.

### C) If category = Other
- Use normal remark/reason flow.

---

## 7) PRINT SIF
### Available actions
- Print single student SIF
- Download all pre-filled SIF (ZIP)

### What gets included
- Attendance call records
- Poor result call records
- Direct call records

---

## 8) Reports
- `Attendance Report`: attendance call summary.
- `Result Report`: result call summary.
- WhatsApp-style summary text is available where configured.

---

## 9) Mobile Number Updates
- In Mentor SIF/student list, mentor can update Student/Father number.
- Updates reflect in coordinator view too.
- Coordinator can identify mentor-updated values visually.

---

## 10) Quick Troubleshooting
- No records visible:
  - Check selected module at top.
  - Check if coordinator uploaded data in that module.
- Wrong students visible:
  - Confirm you are logged into correct mentor identity (full/short name mapping).
- Upload/report delay:
  - Wait for processing to complete.
  - For large result uploads, progress may take a few minutes.

---

## 11) Best Practice Daily Flow
1. Select correct module.
2. Finish `Attendance Calls` (if teaching period).
3. Finish `Result Calls` (if exam period).
4. Use `Direct Calls` for extra call entries.
5. Check `Attendance/Result Report`.
6. Use `PRINT SIF` before physical record update.
7. Buy developer a Chai.
